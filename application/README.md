# Application

The applied half of this repo — see the [root README](../README.md) for
the workshop context and design rationale. Everything below assumes your
working directory is `application/` (this folder), not the repo root.

## Layout

```
scripts/export_for_viz.py    Exports the market-intelligence DB -> data/exports/*.csv + jobmarket.parquet
scripts/generate_sample_data.py  Generates the committed synthetic sample -> data/sample/
viz/shared.py                 Palette, helpers (week_start, normalize_location, classify_rising_cooling, ...)
viz/01_matplotlib_seaborn.py  Salary bands, role composition, top hiring organisations
viz/02_plotly.py              Skill trends, salary by experience, skill co-occurrence heatmap
viz/03_altair.py              Work-model mix over time, top UK locations
viz/04_bokeh.py               Weekly posting volume
viz/dashboard.py              Streamlit app — dark/greyscale/amber "financial terminal" theme
viz/story/                    Scrollytelling page (D3 v7 + Scrollama) — see viz/story/README.md
tests/                        pytest coverage for the story data builder
```

See [`viz/README.md`](viz/README.md) for the full design-system writeup,
real-data caveats, and a chart-by-chart takeaway table. See
[`viz/story/README.md`](viz/story/README.md) for the story's narrative
outline, sourcing, and its own honest cut-corners list.

## Setup

```bash
python -m venv .venv
.venv\Scripts\activate            # PowerShell: .venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

## Zero-setup: running against the committed synthetic sample

No database needed. `--sample` (standalone scripts) or `USE_SAMPLE=1`
(anything run via `streamlit run`, which can't cleanly receive a custom
CLI flag) point every loader at `data/sample/` instead of `data/exports/`:

```bash
python viz/01_matplotlib_seaborn.py --sample   # -> viz/output/01_*.png
python viz/02_plotly.py --sample               # -> viz/output/02_*.html
python viz/03_altair.py --sample               # -> viz/output/03_*.html
python viz/04_bokeh.py --sample                # -> viz/output/04_*.html
USE_SAMPLE=1 streamlit run viz/dashboard.py    # -> http://localhost:8501

python viz/story/build_story_data.py --sample  # -> viz/story/data/story.json
cd viz/story && python -m http.server          # -> http://localhost:8000
```

The sample is ~2,000 synthetic rows generated by
`scripts/generate_sample_data.py` — obviously-fake company names, but
distributions shaped to mirror the real export's documented caveats (see
`viz/README.md`). Regenerate it any time with:

```bash
python scripts/generate_sample_data.py
```

## Getting the real data

Both deliverables normally read from `data/exports/`, which is never
committed (see `.gitignore`). Regenerate it from a Postgres connection
string:

```bash
# PowerShell
$env:JOBFORGE_VIZ_DB_URL = "postgresql://..."
python scripts/export_for_viz.py
```

The story additionally needs its own database for
`viz/story/build_story_data.py` (real mode, i.e. without `--sample`) — see
`viz/story/README.md` for the full two-source explanation and the
`JOBFORGE_PIPELINE_DB_URL` env var it expects.

## Running each chart against real data

```bash
python viz/01_matplotlib_seaborn.py   # -> viz/output/01_*.png
python viz/02_plotly.py               # -> viz/output/02_*.html

```
